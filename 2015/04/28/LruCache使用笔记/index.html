LruCache特点

LruCache是android sdk提供的一种RAM缓存解决方案，这里是官方的说明文档。从文档了解到LruCache的主要特点有：

  • 内部使用一个队列管理数据，最近访问的数据将被移到队列头部
  • 如果往已满的LruCache里添加数据，队列尾部的数据将被移除
  • 线程安全
  • 可自定义cache占用空间的计算方法
  • cache中的key和value都不允许null对象

    内部实现

    我比较关心的是其内部是怎么实现最近被访问的数据被移动到队列的头部（Each time a value is accessed, it is moved to the head of a queue.）
    LruCache的内部实现比较简单，整个类包含注释还不到400行。

内部由一个LinkedHashMap来管理数据，其他的几个变量都是用来辅助管理的。接下来一个一个的看对外提供的接口。

构造函数

构造函数做了两件事情，初始化maxSize，初始化map对象，LinkedHashMap的构造函数的三个参数的含义：

public LinkedHashMap(int initialCapacity,
float loadFactor,
boolean accessOrder)
Constructs an empty LinkedHashMap instance with the specified initial capacity, load factor and ordering mode.
Parameters:
initialCapacity - the initial capacity
loadFactor - the load factor
accessOrder - the ordering mode - true for access-order, false for insertion-order
Throws:
IllegalArgumentException - if the initial capacity is negative or the load factor is nonpositive

这里第三个参数 accessOrder - the ordering mode - true for access-order, false for insertion-order我理解为true的情况下hashmap就按照访问顺序排序，反之则按照插入顺序排序，LruCache通过构造map时，设置accessOrder为true达到最近访问的数据被移到队列头部的目的

数据获取 get（）

数据获取的函数实现如下：

首先访问map，根据访问结果更新hitCount 或者missCount。
如果map中没有key对应的值，则会尝试根据这个key创建一个value（需要使用者重载 create(K)函数）。创建完成后尝试将新创建的value添加到map，此时有可能map中又有了这个key对应的value，则放弃心新创建的value，并返回map中的value。如果新创建的value成功添加到map，则和put一样，需要检查cache的大小trimToSize(maxSize)
