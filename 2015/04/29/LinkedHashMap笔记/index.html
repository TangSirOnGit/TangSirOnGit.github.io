在LruCache使用笔记里面说道，LruCache内部通过LinkedHashMap管理数据，实现“最近访问的数据被移动到队列的头部”。今天就来看一下LinkedHashMap是怎么实现的。LinkedHashMap详细说明

数据成员

LinkedHashMap继承于HashMap,有成员变量

第一个变量header，注释已经做了详细的说明，header就是map中key-value对组成的双向链表中的一个空对象。第一个真实的key-value对是header.nxt,最后一个key-value对是header.prv如果header.nxt == header && header.prv == header那么这个哈希表是空的。第二个成员变量accessOrder就是用来控制双向链表的排序方式的。

这里是通过map.entrySet()迭代来遍历所有元素的，那么LinkedHaspMap是怎样保证map.entrySet()遍历的顺序就是和他的双向链表顺序是一样的呢？

LinkedHashMap实现了Iterator接口LinkedHashIterator,key, value 和Entry迭代器都继承自LinkedHashIterator,各自的next()函数返回值都来自nextEntry()，而nextEntry()就是以header.nxt为起点，按照双向链表的顺序进行遍历。
